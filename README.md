## 📖 项目简介

**IELOCK** 是一款基于 UDP 协议的 Minecraft Java 版服务器压力测试（“锁服”）工具，模拟大量恶意握手请求，用于评估服务器在高负载下的稳定性和抗压能力。  
本项目为 **Android 客户端**，以 APK 形式在移动设备上运行，便于随时随地测试。代码采用 Java 7 编写，兼容 Android 4.4+，完全开源。

本项目协议逻辑与 Python 原版脚本 **完全一致**，可作为便携式替代方案。

## ✨ 主要功能

- **自定义目标**：输入目标服务器 IP 和端口（默认为 25565）。
- **一键启动/停止**：点击“锁服”开始无限循环攻击，点击“停止”立即终止。
- **实时日志显示**：滚动日志显示握手状态、崩溃次数、恢复等待等完整过程。
- **自动重连机制**：服务器宕机后自动探测恢复状态并继续下一轮攻击。
- **免责声明弹窗**：首次启动时强制显示，确保用户知悉使用条款。

🚀 使用方法

1. 打开 App，阅读并同意免责声明。
2. 输入目标 IP 和端口（如 192.168.1.100、25565）。
3. 点击 “锁服” 开始测试。
4. 观察日志输出，程序会循环执行：
   · 握手（Handshake）→ 发送 NACK 包 → 等待 5 秒 → 探测服务器存活状态。
   · 若服务器宕机，则等待恢复（最长 10 秒）并记录崩溃次数。
   · 若连续 2 次恢复失败，则自动退出循环。
5. 再次点击 “停止” 可随时终止。

🔧 核心实现原理

· 模拟 Minecraft 握手协议（UDP 包序列 0x05 → 0x06 → 0x07 → 0x08 → 0x84 等）。
· 使用 DatagramSocket 收发 UDP 数据包，ByteBuffer / byte[] 组装二进制协议。
· 多线程架构：UI 线程与后台攻击线程分离，保证界面流畅。
· 循环控制与状态机：包含握手失败重试、崩溃计数、等待恢复等逻辑。
